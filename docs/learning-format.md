# Формат обучения

## Теория и практика

Каждая тема начинается с короткой теории, которая помогает понять, что именно делает Automation QA engineer и почему выбранный подход важен.

## Достаточно теории для понимания и собеседований

Мы не уходим глубоко во внутренние детали runtime или browser engine. Теории должно хватать, чтобы уверенно объяснить тему на собеседовании и применять ее в работе.

## Фокус на «как» и «почему»

Курс делает акцент на вопросах “how” и “why”: как написать test, почему locator должен быть стабильным, зачем нужен Page Object Model и когда использовать fixture.

## Практические упражнения после каждой темы

После каждой важной темы студент выполняет небольшое практическое упражнение, чтобы сразу закрепить материал.

## Пошаговое обучение

Материалы идут пошагово: от настройки окружения и Git до Playwright, API testing, CI и финального проекта.

## Примеры кода

Примеры кода будут короткими и сфокусированными на QA-задачах, а не на абстрактных примерах программирования.

## Небольшие реальные QA-сценарии

В каждом модуле постепенно добавляются небольшие реальные QA-сценарии: проверка формы, login flow, assertions, test data и анализ ошибок.

## Вопросы для проверки понимания

Вопросы в конце модуля помогают проверить, понял ли студент идею, trade-offs и типичные ошибки.

## Домашнее задание и ревью

Домашнее задание выполняется в отдельной ветке (branch), отправляется через pull request и проверяется по чеклисту. Ревьюер смотрит не только результат, но и понятность решения.
