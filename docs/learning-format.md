# Learning Format

## Theory and practice

Каждая тема начинается с короткой theory, которая помогает понять, что именно делает Automation QA engineer и почему выбранный подход важен.

## Enough theory for understanding and interviews

Мы не уходим глубоко во внутренние детали runtime или browser engine. Теории должно хватать, чтобы уверенно объяснить тему на interview и применять ее в работе.

## Focus on how and why

Курс делает акцент на вопросах “how” и “why”: как написать test, почему locator должен быть стабильным, зачем нужен Page Object Model и когда использовать fixture.

## Practical exercises after each topic

После каждой важной темы студент выполняет небольшое practical exercise, чтобы сразу закрепить материал.

## Step-by-step guidance

Материалы идут пошагово: от environment setup и Git до Playwright, API testing, CI и final project.

## Code examples

Code examples будут короткими и сфокусированными на QA-задачах, а не на абстрактных programming examples.

## Small real-life QA use cases

В каждом module постепенно добавляются маленькие real-life QA use cases: проверка формы, login flow, assertions, test data и анализ failures.

## Questions to verify understanding

Вопросы в конце module помогают проверить, понял ли студент идею, trade-offs и типичные ошибки.

## Homework and review process

Homework выполняется в отдельном branch, отправляется через pull request и проверяется по checklist. Reviewer смотрит не только результат, но и понятность решения.
